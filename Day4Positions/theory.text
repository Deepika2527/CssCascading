CSS POSITION PROPERTY - COMPLETE THEORY NOTES
====================================================

1. WHAT IS POSITION IN CSS?
----------------------------------------------------

The position property in CSS determines how an element is placed on a webpage.

It controls:
1. Where the element appears.
2. How it behaves in the document flow.
3. Whether it moves during scrolling.
4. Which reference point (parent, viewport, current position) it follows.

Syntax:

position: value;

Example:

position: static;
position: relative;
position: absolute;
position: fixed;
position: sticky;

----------------------------------------------------
2. TYPES OF POSITION
----------------------------------------------------

There are 5 types of position values:

1. static
2. relative
3. absolute
4. fixed
5. sticky

----------------------------------------------------
3. POSITIONING PROPERTIES
----------------------------------------------------

The following properties are used along with position:

top
right
bottom
left

Example:

.box{
    position: relative;
    top: 20px;
    left: 30px;
}

Direction Understanding:

top: 20px;
Moves the element 20px downward.

left: 20px;
Moves the element 20px right.

right: 20px;
Moves the element 20px left.

bottom: 20px;
Moves the element 20px upward.

----------------------------------------------------
4. POSITION: STATIC
----------------------------------------------------

Definition:

Static is the default position value for every HTML element.

Syntax:

position: static;

Characteristics:

1. Default position.
2. Follows normal document flow.
3. Ignores top, right, bottom, and left.
4. Cannot be repositioned using offset properties.

Example:

.box{
    position: static;
    top: 20px;
}

Result:

top:20px will not work because static elements ignore positioning properties.

Use Case:

Used when no special positioning is required.

----------------------------------------------------
5. POSITION: RELATIVE
----------------------------------------------------

Definition:

Relative positioning moves an element relative to its original position.

Syntax:

position: relative;

Characteristics:

1. Remains in normal document flow.
2. Original space remains reserved.
3. Supports top, right, bottom, and left.
4. Moves relative to its current position.

Example:

.box{
    position: relative;
    top: 20px;
    left: 30px;
}

Result:

The element moves:
- 20px down
- 30px right

The original space remains occupied.

Use Cases:

1. Small adjustments to element positions.
2. Creating a positioning context for absolute elements.

----------------------------------------------------
6. POSITION: ABSOLUTE
----------------------------------------------------

Definition:

Absolute positioning removes the element from the normal document flow and positions it relative to its nearest positioned ancestor.

Syntax:

position: absolute;

Characteristics:

1. Removed from document flow.
2. No space is reserved.
3. Supports top, right, bottom, and left.
4. Searches for the nearest positioned parent.

Positioned Parent Means:

A parent having:

position: relative;
position: absolute;
position: fixed;
position: sticky;

Example:

.parent{
    position: relative;
}

.child{
    position: absolute;
    top: 20px;
    left: 20px;
}

Result:

The child element is positioned relative to the parent.

Important Rule:

If no positioned parent exists,
the absolute element uses the viewport/body as reference.

Use Cases:

1. Badges
2. Icons
3. Overlays
4. Tooltips
5. Custom layouts

----------------------------------------------------
7. POSITION: FIXED
----------------------------------------------------

Definition:

Fixed positioning attaches an element to the browser viewport.

Syntax:

position: fixed;

Characteristics:

1. Removed from document flow.
2. No space reserved.
3. Positioned relative to viewport.
4. Does not move while scrolling.
5. Always remains visible at specified location.

Example:

.box{
    position: fixed;
    bottom: 20px;
    right: 20px;
}

Result:

The element stays at the same position even when the page is scrolled.

Use Cases:

1. WhatsApp Button
2. Chat Support Button
3. Scroll To Top Button
4. Fixed Navigation Bar

----------------------------------------------------
8. POSITION: STICKY
----------------------------------------------------

Definition:

Sticky positioning combines relative and fixed behavior.

Syntax:

position: sticky;

Characteristics:

1. Initially behaves like relative.
2. Becomes fixed after reaching a threshold.
3. Requires top, left, right, or bottom.
4. Remains within parent boundaries.

Example:

.navbar{
    position: sticky;
    top: 0;
}

Result:

Before scrolling:
Acts like relative.

After reaching top:
Acts like fixed.

Use Cases:

1. Sticky Navbar
2. Sticky Table Headers
3. Sidebar Menus
4. Section Headings

10. IMPORTANT INTERVIEW POINTS
----------------------------------------------------

Rule 1:

Static ignores top, left, right, and bottom.

Rule 2:

Relative keeps its original space.

Rule 3:

Absolute removes itself from document flow.

Rule 4:

Fixed remains attached to viewport.

Rule 5:

Sticky becomes fixed after reaching threshold.

Rule 6:

Absolute always searches for the nearest positioned parent.

Rule 7:

If no positioned parent exists,
absolute uses the viewport/body.

----------------------------------------------------
11. INTERVIEW QUESTIONS AND ANSWERS
----------------------------------------------------

Q1. What is the default position of every HTML element?

Answer:

position: static;

----------------------------------------------------

Q2. Which position value ignores top, left, right, and bottom?

Answer:

position: static;

----------------------------------------------------

Q3. What is the difference between relative and absolute?

Answer:

Relative:
- Stays in document flow.
- Original space remains.

Absolute:
- Removed from document flow.
- No space reserved.

----------------------------------------------------

Q4. What is a positioned parent?

Answer:

A parent whose position value is:

position: relative;
position: absolute;
position: fixed;
position: sticky;

----------------------------------------------------

Q5. To whom does an absolute element listen?

Answer:

Nearest positioned parent.

If none exists:
Viewport/body.

----------------------------------------------------

Q6. Why do we usually give position: relative to the parent?

Answer:

To make absolute child elements position themselves relative to that parent.

----------------------------------------------------

Q7. Which position is commonly used for chat buttons?

Answer:

position: fixed;

----------------------------------------------------

Q8. Which position is commonly used for sticky navigation bars?

Answer:

position: sticky;

----------------------------------------------------

Q9. Does relative remove the element from document flow?

Answer:

No.

----------------------------------------------------

Q10. Does absolute remove the element from document flow?

Answer:

Yes.

----------------------------------------------------

Q11. Can sticky work without top property?

Answer:

No.

Example:

position: sticky;
top: 0;

----------------------------------------------------

Q12. What is the difference between fixed and sticky?

Answer:

Fixed:
Always attached to viewport.

Sticky:
Initially behaves like relative and becomes fixed after reaching threshold.

----------------------------------------------------

12. ONE-LINE REVISION
----------------------------------------------------

Static  -> Default position.
Relative -> Moves from current position.
Absolute -> Moves from nearest positioned parent.
Fixed -> Fixed to viewport.
Sticky -> Relative + Fixed behavior.

Relative keeps space.
Absolute removes space.
Fixed ignores scrolling.
Sticky becomes fixed after threshold.
